INSTALLATION
- Décompresse le ZIP avec n'importe quoi dans un seul dossier.
- Félicitation, tu viens d'installer PHORTAIL avec succès !
- Attention ! Pour que le script fonctionne, aucun des fichiers ne doit être en lecture ou écriture seule. Tu ne dois renommer, ni supprimer manuellement les fichiers présents.
- Balance tous les fichiers sur le FTP de ton site avec un simple client FTP (IE, Netscape, CuteFTP, etc). Tu peux bien sûr utiliser PHORTAIL sur un serveur local (PHORTAIL a été créé sur un serveur QuickPHP 1.0 Apache avec PHP 4.0).

COMPATIBILITE AVEC PHORTAIL 1.1
Tout doit être supprimer. La structure du script et le temps ne me permettent pas de faire un utilitaire d'importation. Toutefois il est possible d'importer les articles. Sur PHORTAIL 1.1. Sur PHORTAIL 1.1, prend les fichiers du dossier /pages et copie les fichiers artbase.txt, artnum.txt, et tous les fichiers commençant par art (art5.txt par exemple). Met les sur PHORTAIL 1.2 dans le dossier /include/articles/.

CREER DES ARTICLES, AJOUTER DES IMAGES
Comme le code HTML est autorisé, il est possible d'utiliser tout les tags que l'on veut (IMG, A ...). Il faut savoir que les textes tapés seront entourés des balises "<p></p>".
Pour mettre une image servant de logo ou de fond d'écran il faut la transférer dans le dossier "images". Elle apparaîtra ainsi dans le formulaire du logo et du fond d'écran quand on configure le les couleurs et les noms. Pour les photos, ils faut les placer dans le dossier "photos". TOUT CECI EST OBLIGATOIRE !

FONCTONNEMENT
- Les infos envoyées sur PHORTAIL sont enregistrées dans des fichiers texte dans le dossier include et tous les sous-dossiers. C'est pour cette raison que le caractère | (pipe) sera interdit dans certains formulaires (pour évité une erreur d'interprétation des fichiers texte).
- Le fichier "mdp.php" contient le mot de passe du site. Il ne doit en aucun cas être visible pour les utilisateurs.

ADMINISTRER LE SITE
Il faut se connecter avec le fichier "admin.php". Le mot de passe par défaut est "pass". A partir de là, tout est personnalisable. Je conseille (et même plus, j'oblige ;-) les utilisateurs à changer de mot de passe pour des raisons de sécurité évidentes.
Pour changer le mot de passe, ouvre le fichier mdp.php et modifie la valeur entre les guillemets (les guillemets sont nessessaires).

BUGS REPORT
Ce script a été testé sur Apache + PHP4 et Chez.com. Il n'y a aucune raison que le script plante. Mais si vous vous êtes procuré une version autre que celle distribué sur NSKATE, et que des modifications y ont été apportées, il est possible qu'il y ait quelques plantages si la personne qui a modifié le script n'a pas bien vérifié sa version. Pour des questions, utilise le forum de mon site.
